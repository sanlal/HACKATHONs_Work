# Day 3 development — farmer direct market

Date: July 15, 2026

## Goal

Turn the farmer-market preview into a complete, transparent produce journey
that lets farmers compare direct buyer offers using more than unit price alone.

## What changed

### Interactive farmer experience

- Added farmer and buyer role switching.
- Farmers can create produce listings with crop, variety, quantity, unit,
  grade, harvest date, expected price, location and pickup notes.
- Farmers can see all offers on their own listings.
- Offers are compared using buyer, unit price, benchmark difference, quantity,
  total value, pickup date and terms.
- A farmer can accept exactly one offer.
- Remaining offers are marked rejected after selection.
- A reserved exchange can be marked complete after pickup.
- Produce activity persists in the browser and can be reset.

### Interactive buyer experience

- Buyers can discover open produce listings.
- Buyers can submit quantity, price-per-unit, pickup date and terms.
- The interface calculates total offer value before submission.
- A buyer can update their existing offer instead of creating duplicates.
- Accepted and completed offers remain visible in activity history.

### Transparent benchmark design

- Each listing shows minimum, modal and maximum benchmark values.
- Benchmark market, observation date and source are visible.
- Every included benchmark is explicitly labeled as demonstration data.
- Bid differences are calculated in code and never generated by AI.
- The interface does not call one offer “best”; farmers compare the tradeoffs
  and make the final decision.

### Dashboard

- Added open produce listing counts.
- Added total buyer-offer counts.
- Added completed produce exchange counts to the farmer-market card.
- Farmer market now links to the interactive experience instead of a preview.

### Database and authorization

- Added participant-only bid visibility.
- Added validation for open listings, ownership, quantity, positive price and
  pickup date.
- Added `place_produce_bid` with one-offer-per-buyer upsert behavior.
- Added `accept_produce_bid` with guarded farmer ownership and single-winner
  selection.
- Added `confirm_produce_pickup` with separate farmer and buyer confirmations.
- A completion activity record is created only after both parties confirm in
  Supabase mode.
- Notifications are prepared for new bids, accepted offers and completed
  exchanges.

## Two operating modes

### Interactive demo mode

The current produce UI uses seeded Telangana examples and browser storage. It
works immediately without Supabase credentials and is designed for judging and
video recording.

### Supabase mode

The database migration prepares production-style bid and pickup operations.
After a Supabase project is configured, the UI can be connected to these RPCs:

```text
place_produce_bid
accept_produce_bid
confirm_produce_pickup
```

Run all migrations in filename order:

```text
supabase/migrations/202607140001_initial_schema.sql
supabase/migrations/202607150001_auth_and_workflow.sql
supabase/migrations/202607150002_produce_workflow.sql
```

## How to test Day 3

1. Run `npm install` and `npm run dev`.
2. Open `http://localhost:3000/login`.
3. Select **Enter interactive demo**.
4. Open **Farmer market** from the dashboard.
5. In farmer view, inspect the two seeded paddy offers.
6. Compare total value, unit price, benchmark difference and pickup date.
7. Accept one offer and confirm pickup completion.
8. Select **Reset demo**.
9. In farmer view, create a new produce listing.
10. Switch to buyer view and submit a direct offer on that listing.
11. Switch back to farmer view to compare and accept the new offer.

## Current limitations

- The interactive produce workflow uses browser-persisted demo data. Supabase
  functions are prepared but the UI is not connected to them yet.
- Demonstration benchmark values are fictional and are not live mandi prices.
- The browser demo uses one pickup confirmation for a concise judge flow;
  Supabase mode requires both farmer and buyer confirmation.
- Produce photos, logistics integration and actual payments remain outside the
  current MVP.
- Books remain a preview until Day 4.
- Live GPT-5.6 functionality remains scheduled for Day 5.
